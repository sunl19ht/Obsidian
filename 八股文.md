## 1. Java创建线程的方式
1. 继承Thread类 重写run方法（不是重写start方法）
   缺点：Java是类单继承 接口和接口之间是多继承，如果继承了Thread方法那么就不能继承其他方法
2. 实现Runnable接口 实现run方法（Thread底层也是实现Runnable接口）
3. 实现Callable接口 实现call方法 可以获取任务结果获取返回值
4. 线程池创建线程 ExecutorService
## 2. 为什么不推荐使用Executors创建线程池？
```java
publci static ExcutorService newFixedThreadPool(int nTrheads) {
	return new ThreadPoolExecutor(nThreads, nThreads, 0L, TimeUnit.MILLISECONDS, new LinkedBlockingQueue<Runnaable>());
}
```
1. 创建的队列为LinkedBlockingQueue，是一个无界阻塞队列，如果使用该线程池执行任务，任务过多就会不断的添加到队列当中造成OOM
2. 工作中建议直接使用ThreadPoolExecutor来定义线程池
## 3. 线程池有哪几种状态，分别表示什么意思？
1. RUNNING：
   表示线程池正在执行，既能接受新任务，也会正常处理队列中的任务
2. SHUTDOWN：
   调用线程池shutdown()方法时，线程池会进入SHUTDOWN状态，表示线程池为关闭状态，不会接受新任务，但是会继续把队列中的任务处理完
3. STOP：
   调用线城市shutdownnow()方法时，线程池进入STOP状态，表示线程池处处于停止状态，此状态下线程池不会接受新任务，也不会处理队列中的任务，并且正在运行的线程也会中断
4. TIDYING：
   线程池中没有线程在运行后，线程池的状态会自动变为TIDYING状态，并且会调用terminated()，该方法是空方法，留给程序员进行扩展
5. TERMINATED：
   terminated()方法执行完之后，线程池状态会进入TERMINATED
## 4. Sychronized和ReentrantLock有哪些不同点？

| sychronized      | ReentrantLock        |
| ---------------- | -------------------- |
| Java的关键字         | JDK提供的类              |
| 自动加锁与自动释放锁       | 需要手动加锁与释放锁           |
| JVM层面的锁          | API层面的锁              |
| 非公平锁             | 公平锁或非公平锁             |
| 锁的是对象，锁信息保存在对象头中 | int类型的state标识来标识锁的状态 |
| 底层有锁升级过程         | 没有锁升级过程              |
## 5. JDK，JRE，JVM之间的区别
1. JDK：Java标准开发包，Java编译器，Java运行时环境，以及常用的Java类库
2. JRE：Java运行环境，用于运行Java字节码文件
3. JVM：Java虚拟机，是JRE的一部分，负责运行字节码文件
