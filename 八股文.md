## 1. Java创建线程的方式
1. 继承Thread类 重写run方法（不是重写start方法）
   缺点：Java是类单继承 接口和接口之间是多继承，如果继承了Thread方法那么就不能继承其他方法
2. 实现Runnable接口 实现run方法（Thread底层也是实现Runnable接口）
3. 实现Callable接口 实现call方法 可以获取任务结果获取返回值
4. 线程池创建线程 ExecutorService
## 2. 为什么不推荐使用Executors创建线程池？
```java
publci static ExcutorService newFixedThreadPool(int nTrheads) {
	return new ThreadPoolExecutor(nThreads, nThreads, 0L, TimeUnit.MILLISECONDS, new LinkedBlockingQueue<Runnaable>());
}
```
1. 创建的队列为LinkedBlockingQueue，是一个无界阻塞队列，如果使用该线程池执行任务，任务过多就会不断的添加到队列当中造成OOM
2. 工作中建议直接使用ThreadPoolExecutor来定义线程池
## 3. 线程池有哪几种状态，分别表示什么意思？
1. RUNNING：
   表示线程池正在执行，既能接受新任务，也会正常处理队列中的任务
2. SHUTDOWN：
   调用线程池shutdown()方法时，线程池会进入SHUTDOWN状态，表示线程池为关闭状态，不会接受新任务，但是会继续把队列中的任务处理完
3. STOP：
   调用线城市shutdownnow()方法时，线程池进入STOP状态，表示线程池处处于停止状态，此状态下线程池不会接受新任务，也不会处理队列中的任务，并且正在运行的线程也会中断
4. TIDYING：
   线程池中没有线程在运行后，线程池的状态会自动变为TIDYING状态，并且会调用terminated()，该方法是空方法，留给程序员进行扩展
5. TERMINATED：
   terminated()方法执行完之后，线程池状态会进入TERMINATED

## 5. JDK，JRE，JVM之间的区别
1. JDK：Java标准开发包，Java编译器，Java运行时环境，以及常用的Java类库
2. JRE：Java运行环境，用于运行Java字节码文件
3. JVM：Java虚拟机，是JRE的一部分，负责运行字节码文件
## 6. hashCode()与equals()方法之间的关系
1. 两个对象hashCode不相同那么这两个对象肯定不是同一个对象
2. 如果两个对象的hashCode相同，不代表这两个对象一定是同一个对象，也有可能是两个
3. 如果两个对象相同，那么他们的hashCode一定相同
## 7. String，StringBuffer，StringBuilder区别

| **特性**   | **String**           | **StringBuffer**              | **StringBuilder**  |
| -------- | -------------------- | ----------------------------- | ------------------ |
| **可变性**  | **不可变**（Immutable）   | **可变**（Mutable）               | **可变**（Mutable）    |
| **线程安全** | **安全**（由于不可变，天然线程安全） | **安全**（方法加了 `synchronized` 锁） | **不安全**（无锁）        |
| **性能**   | 低（频繁修改会创建大量新对象）      | 中等（保证了线程安全，有锁开销）              | **高**（没有锁的开销，速度最快） |
| **引入版本** | JDK 1.0              | JDK 1.0                       | JDK 1.5            |

## 9. == 和equals()的区别
1. == 是基本数据类型是比较值，如果是引用类型，比较的是引用地址
2. equals 具体看各类重写equals方法之后的比较逻辑，比如String类，虽然是引用类型，但是String重写了equals方法，方法内部是比较字符串的各个字符全部相等
## 10. 重载和重写的区别
1. 重载：发生在同一个方法中，必须方法名相同，参数类型不同，个数不同，顺序不同，方法返回值和访问修饰符可以不同，发生在编译时
2. 重写：发生在父子类中，方法名，参数列表必须相同，返回值范围小于等于父类，抛出的异常范围小于等于父类，访问修饰符范围大于父类；如果父类方法访问修饰符为private则子类不能重写该方法
## 11. List和Set的区别
1. List：有序（按照对象进入顺序保存对象），可重复，允许多个Null元素对象，可以使用iterator迭代器取出所有元素，还可以使用get(int index)获取指定下标元素
2. Set：无序，不可重复，最多允许有一个Null对象，取出元素只能使用iterator接口获取所有元素
## 12. ArrayList和LinkedList区别
1. 首先底层结构不同，ArrayList基于数组实现，更适合随机查找。LinkedList基于链表实现，更适合删除和添加，查询，添加，删除，的时间复杂度不同。
2. ArrayList和LinkedList都实现了List接口，但LinkedList实现了Deque接口，可以当作队列使用
## 13. JDK1.7到JDK1.8 HashMap发生了什么变化
1. 1.7中底层是数组+链表，1.8中是数组+链表+红黑树，加红黑树的目的是提高HashMap插入和查询的效率
2. 1.7链表插入使用的是头插法，1.8是尾插法，因为1.8中插入key和value时需要判断链表元素个数，所以需要遍历链表统计元素个数，所以正好就直接使用尾插法
3. 1.7中哈希算法比较复杂，存在各种右移与异或运算，1.8中进行了简化，因为复杂的哈希算法目的就是提高散列性，来提供HashMap的整体效率，而1.8中新增了红黑树，所以可以适当的简化哈希算法节省CPU资源
## 14. 线上OOM怎么定位和解决
1. 一次性申请太多资源 更改申请对象数量
2. 内存资源耗尽未释放 找到未释放的资源
3. 本身资源不够 用jmap -heap查看堆信息
## 15. 订单超时如何实现
1. JDK自带的延时队列 DelayQueue 新增订单会把订单插入到DelayQueue，按照订单时间排序，单独起一个线程轮询订单的头部，如果超时就进行处理，在数据库中修改订单状态
   优点：简单，不需要借助第三方组件，成本低
   缺点：所有超时订单都要加入DelayQueue中，占用内存大，不适合订单大的场景和分布式系统
2. RocketMQ 新增订单Producer发送定时消息到MQ，额外指定时间，MQ到达指定时间将消息发送到Consumer更新订单状态
   优点：使用简单，支持分布式，精度高支持任意时间
   缺点：订单最大时长为24小时，成本高，每个订单都需要产生一个消息到MQ中占用存储成本，同一时间出现大量消息会影响系统性能
3. Redis过期监听，单独对Redis开启过期监听，新增订单添加一个订单对应的过期key，过期时间到之后redis过期时间监听处理订单状态
   缺点：不可靠，redis过期时应用正好重启呢么事件就丢了，会导致订单无法关闭，如果订单量大也会占用大量的存储空间
